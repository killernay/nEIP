## 2. Quote-to-Cash (ขายสินค้า → รับเงิน)

### สถานการณ์ตัวอย่าง

> **บริษัท ตัวอย่าง จำกัด** ขายสินค้า Product A จำนวน 10 ชิ้น ราคาชิ้นละ 1,000 บาท
> ให้ลูกค้า **บริษัท ลูกค้าดี จำกัด** เครดิต 30 วัน

### ขั้นตอนที่ 1: สร้างใบเสนอราคา (Quotation)

| รายการ | รายละเอียด |
|--------|-----------|
| **ผู้ใช้ทำ** | สร้างใบเสนอราคาผ่านหน้า Sales → Quotation → New |
| **API** | `POST /api/v1/quotations` |
| **ข้อมูลที่ส่ง** | ลูกค้า, รายการสินค้า 10 x 1,000 THB |
| **ผลลัพธ์** | สร้าง Quotation QT-2026-0001, status = `draft` |
| **ตารางที่เปลี่ยน** | `quotations`, `quotation_line_items` |
| **GL Entry** | ไม่มี (ใบเสนอราคาไม่กระทบบัญชี) |

```
สินค้า: Product A — สินค้าตัวอย่าง
จำนวน: 10 ชิ้น × 1,000.00 บาท = 10,000.00 บาท (subtotal)
VAT 7%:                            700.00 บาท
รวมทั้งสิ้น (Grand Total):         10,700.00 บาท
```

### ขั้นตอนที่ 2: แปลงเป็นใบสั่งขาย (Sales Order)

| รายการ | รายละเอียด |
|--------|-----------|
| **ผู้ใช้ทำ** | กดปุ่ม "Convert to SO" จากใบเสนอราคา |
| **API** | `POST /api/v1/sales-orders` |
| **ผลลัพธ์** | สร้าง Sales Order SO-2026-0001, status = `confirmed` |
| **ตารางที่เปลี่ยน** | `sales_orders`, `sales_order_line_items` |
| **GL Entry** | ไม่มี (ใบสั่งขายไม่กระทบบัญชี) |

### ขั้นตอนที่ 3: ส่งสินค้า (Delivery Order)

| รายการ | รายละเอียด |
|--------|-----------|
| **ผู้ใช้ทำ** | สร้างใบส่งสินค้า → ยืนยันการส่ง |
| **API** | `POST /api/v1/delivery-orders` → `POST /api/v1/delivery-orders/:id/confirm` |
| **ผลลัพธ์** | DO-2026-0001, status = `delivered` |
| **ตารางที่เปลี่ยน** | `delivery_orders`, `stock_movements`, `stock_levels` |
| **ผลต่อ Inventory** | Product A: quantity_on_hand ลดลง 10 ชิ้น |

```
stock_movements:
  product_id: Product A
  warehouse_id: WH-001
  movement_type: 'issue'
  quantity: 10
  reference_type: 'delivery_order'
```

### ขั้นตอนที่ 4: ออกใบแจ้งหนี้/ใบกำกับภาษี (Invoice)

| รายการ | รายละเอียด |
|--------|-----------|
| **ผู้ใช้ทำ** | สร้าง Invoice → Post (เปลี่ยน draft → posted) |
| **API** | `POST /api/v1/invoices` → `POST /api/v1/invoices/:id/post` |
| **ผลลัพธ์** | INV-2026-0001, status = `posted`, สร้าง Journal Entry อัตโนมัติ |
| **ตารางที่เปลี่ยน** | `invoices`, `invoice_line_items`, `journal_entries`, `journal_entry_lines` |

#### GL Entry เมื่อ Post Invoice (บันทึกรายได้)

```
┌─────────────────────────────────────────────────────────────────┐
│  Journal Entry: JE-2026-0001                                    │
│  Description: "Invoice posted: INV-2026-0001"                   │
│  Status: posted                                                 │
├─────┬────────────────────────────┬──────────┬───────────────────┤
│ Line│ บัญชี (Account)            │ Dr (เดบิต)│ Cr (เครดิต)       │
├─────┼────────────────────────────┼──────────┼───────────────────┤
│  1  │ 1100 ลูกหนี้การค้า (AR)     │ 10,700   │                   │
│  2  │ 4100 รายได้จากการขาย        │          │ 10,000            │
│  3  │ 2110 ภาษีขายค้างจ่าย        │          │    700            │
│     │      (VAT Payable)         │          │                   │
├─────┼────────────────────────────┼──────────┼───────────────────┤
│     │ รวม                        │ 10,700   │ 10,700            │
└─────┴────────────────────────────┴──────────┴───────────────────┘
```

**หลักการ:** Dr AR (grandTotal = subtotal + VAT 7%) / Cr Revenue (subtotal) + Cr VAT Payable (7%)

> **สำคัญ:** ระบบคำนวณ VAT จาก subtotal ด้วย bigint arithmetic:
> `vatAmount = subtotal × 700 / 10000` (round half up)
> เก็บค่าเป็น satang (สตางค์) เพื่อความแม่นยำ: 10,000 บาท = 1,000,000 satang

### ขั้นตอนที่ 5: ออกใบกำกับภาษีอิเล็กทรอนิกส์ (e-Tax Invoice)

| รายการ | รายละเอียด |
|--------|-----------|
| **ผู้ใช้ทำ** | ดู e-Tax Invoice จาก Invoice ที่ posted แล้ว |
| **API** | `GET /api/v1/invoices/:id/e-tax` |
| **ผลลัพธ์** | JSON e-Tax Invoice ตามมาตรฐานกรมสรรพากร (documentTypeCode: T02) |

```json
{
  "documentType": "ใบกำกับภาษี",
  "documentTypeCode": "T02",
  "seller": { "companyName": "บริษัท ตัวอย่าง จำกัด", "taxId": "0105XXXXXXXXX" },
  "buyer":  { "companyName": "บริษัท ลูกค้าดี จำกัด", "taxId": "0105YYYYYYYYY" },
  "invoice": {
    "subTotalSatang": "1000000",
    "vatAmountSatang": "70000",
    "grandTotalSatang": "1070000",
    "currency": "THB"
  }
}
```

### ขั้นตอนที่ 6: รับชำระเงิน (Payment Receipt)

| รายการ | รายละเอียด |
|--------|-----------|
| **ผู้ใช้ทำ** | บันทึกการรับเงินโอน 10,700 บาท |
| **API** | `POST /api/v1/payments` (with invoiceId) |
| **ผลลัพธ์** | PMT-2026-0001, status = `matched`, Invoice status → `paid` |
| **ตารางที่เปลี่ยน** | `ar_payments`, `invoice_payments`, `invoices`, `journal_entries`, `journal_entry_lines` |

#### GL Entry เมื่อรับเงิน (บันทึกการรับชำระ)

```
┌─────────────────────────────────────────────────────────────────┐
│  Journal Entry: JE-2026-0002                                    │
│  Description: "AR payment received - PMT-2026-0001"             │
│  Status: posted                                                 │
├─────┬────────────────────────────┬──────────┬───────────────────┤
│ Line│ บัญชี (Account)            │ Dr (เดบิต)│ Cr (เครดิต)       │
├─────┼────────────────────────────┼──────────┼───────────────────┤
│  1  │ 1010 เงินสด/ธนาคาร (Cash)  │ 10,700   │                   │
│  2  │ 1100 ลูกหนี้การค้า (AR)     │          │ 10,700            │
├─────┼────────────────────────────┼──────────┼───────────────────┤
│     │ รวม                        │ 10,700   │ 10,700            │
└─────┴────────────────────────────┴──────────┴───────────────────┘
```

**หลักการ:** Dr Cash/Bank / Cr AR — ล้างยอดลูกหนี้

### สรุป Quote-to-Cash Flow

```

ใบเสนอราคา  ──▶  ใบสั่งขาย  ──▶  ใบส่งสินค้า  ──▶  ใบแจ้งหนี้  ──▶  รับชำระเงิน
(draft)         (confirmed)     (delivered)      (posted)         (matched)
  ไม่มี JE        ไม่มี JE       -Stock          Dr AR/Cr Rev     Dr Cash/Cr AR
                                                +Cr VAT
```

## 3. Procure-to-Pay (ซื้อสินค้า → จ่ายเงิน)

### สถานการณ์ตัวอย่าง

> **บริษัท ตัวอย่าง จำกัด** ซื้อวัตถุดิบ Material B จำนวน 50 ชิ้น ราคาชิ้นละ 500 บาท
> จาก **บริษัท ซัพพลายเออร์ จำกัด** เครดิต 45 วัน, หัก ณ ที่จ่าย 3%

### ขั้นตอนที่ 1: สร้างใบขอซื้อ (Purchase Requisition)

| รายการ | รายละเอียด |
|--------|-----------|
| **ผู้ใช้ทำ** | แผนกคลังสร้าง PR เมื่อสต็อกต่ำ |
| **API** | `POST /api/v1/purchase-requisitions` |
| **ผลลัพธ์** | PR-2026-0001, status = `pending_approval` |
| **GL Entry** | ไม่มี |

```
สินค้า: Material B — วัตถุดิบตัวอย่าง
จำนวน: 50 ชิ้น × 500.00 บาท = 25,000.00 บาท
```

### ขั้นตอนที่ 2: ขอใบเสนอราคา (RFQ) และเปรียบเทียบ

| รายการ | รายละเอียด |
|--------|-----------|
| **ผู้ใช้ทำ** | ส่ง RFQ ให้ผู้ขาย 3 ราย → เลือกรายที่ดีที่สุด |
| **API** | `POST /api/v1/rfqs` |
| **ผลลัพธ์** | RFQ-2026-0001, เลือก บริษัท ซัพพลายเออร์ จำกัด |
| **GL Entry** | ไม่มี |

### ขั้นตอนที่ 3: สร้างใบสั่งซื้อ (Purchase Order)

| รายการ | รายละเอียด |
|--------|-----------|
| **ผู้ใช้ทำ** | แปลง PR → PO, ส่งให้ผู้ขาย |
| **API** | `POST /api/v1/purchase-orders` |
| **ผลลัพธ์** | PO-2026-0001, status = `confirmed` |
| **GL Entry** | ไม่มี (commitment เท่านั้น) |

### ขั้นตอนที่ 4: รับสินค้า (Goods Receipt)

| รายการ | รายละเอียด |
|--------|-----------|
| **ผู้ใช้ทำ** | ตรวจรับสินค้า 50 ชิ้น ที่คลังสินค้า |
| **API** | `POST /api/v1/goods-receipts` |
| **ผลลัพธ์** | GR-2026-0001, status = `received` |
| **ตารางที่เปลี่ยน** | `goods_receipts`, `stock_movements`, `stock_levels` |
| **ผลต่อ Inventory** | Material B: quantity_on_hand เพิ่ม 50 ชิ้น |

```
stock_movements:
  product_id: Material B
  warehouse_id: WH-001
  movement_type: 'receive'
  quantity: 50
  reference_type: 'goods_receipt'
```

### ขั้นตอนที่ 5: รับบิลจากผู้ขาย (Vendor Bill) และ Post

| รายการ | รายละเอียด |
|--------|-----------|
| **ผู้ใช้ทำ** | บันทึกบิลจากผู้ขาย → Post |
| **API/Tool** | `ap.createBill` → `ap.postBill` |
| **ผลลัพธ์** | BILL-2026-0001, status = `posted`, สร้าง JE อัตโนมัติ |
| **ตารางที่เปลี่ยน** | `bills`, `bill_line_items`, `journal_entries`, `journal_entry_lines` |

#### GL Entry เมื่อ Post Bill (บันทึกค่าใช้จ่าย)

```
┌─────────────────────────────────────────────────────────────────┐
│  Journal Entry: JE-2026-0003                                    │
│  Description: "Bill posted: BILL-2026-0001"                     │
│  Status: posted                                                 │
├─────┬────────────────────────────┬──────────┬───────────────────┤
│ Line│ บัญชี (Account)            │ Dr (เดบิต)│ Cr (เครดิต)       │
├─────┼────────────────────────────┼──────────┼───────────────────┤
│  1  │ 5100 ต้นทุนสินค้า (COGS)   │ 25,000   │                   │
│  2  │ 1170 ภาษีซื้อ (Input VAT)  │  1,750   │                   │
│  3  │ 2100 เจ้าหนี้การค้า (AP)    │          │ 26,750            │
├─────┼────────────────────────────┼──────────┼───────────────────┤
│     │ รวม                        │ 26,750   │ 26,750            │
└─────┴────────────────────────────┴──────────┴───────────────────┘
```

**หลักการ:** Dr Expense per line (subtotal) + Dr Input VAT 7% / Cr AP (grandTotal)

> **การคำนวณ:**
> Subtotal = 50 × 500 = 25,000 บาท
> VAT 7% = 25,000 × 7% = 1,750 บาท
> Grand Total = 25,000 + 1,750 = 26,750 บาท

### ขั้นตอนที่ 6: Three-Way Match (จับคู่ 3 ทาง)

| รายการ | รายละเอียด |
|--------|-----------|
| **ระบบทำ** | ตรวจสอบ PO ↔ GR ↔ Bill ตรงกัน |
| **ตรวจสอบ** | จำนวน, ราคา, ผู้ขาย ตรงกัน 3 เอกสาร |
| **ผลลัพธ์** | Match status = `matched` → พร้อมจ่ายเงิน |

```
Three-Way Match:
  PO-2026-0001:   50 × 500 = 25,000  ✓ ตรงกัน
  GR-2026-0001:   50 ชิ้นรับครบ        ✓ ตรงกัน
  BILL-2026-0001: 50 × 500 = 25,000  ✓ ตรงกัน
  → Status: MATCHED ✅
```

## 6. Inventory Cycle (สินค้าคงคลัง)

### สถานการณ์ตัวอย่าง

> คลังสินค้า WH-001 มี Product A:
> - รับเข้า 100 ชิ้น (จากการซื้อ)
> - ขายออก 30 ชิ้น
> - ยอดตามบัญชี = 70 ชิ้น
> - ตรวจนับจริงพบ 68 ชิ้น (ขาด 2 ชิ้น = variance)

### ขั้นตอนที่ 1: รับสินค้าเข้าคลัง (Goods Receipt)

```
stock_movements:
  movement_type: 'receive'
  quantity: 100
  balance_after: 100

stock_levels:
  product_id: Product A
  warehouse_id: WH-001
  quantity_on_hand: 100
  quantity_available: 100
```

### ขั้นตอนที่ 2: เบิก/ขายสินค้า (Issue)

```
stock_movements:
  movement_type: 'issue'
  quantity: 30
  balance_after: 70

stock_levels:
  quantity_on_hand: 70
  quantity_available: 70
```

### ขั้นตอนที่ 3: โอนย้ายระหว่างคลัง (Transfer)

```
stock_movements (คลังต้นทาง WH-001):
  movement_type: 'transfer_out'
  quantity: 10

stock_movements (คลังปลายทาง WH-002):
  movement_type: 'transfer_in'
  quantity: 10
```

### ขั้นตอนที่ 4: ตรวจนับสต็อก (Stock Count)

| ขั้นตอน | API | ผลลัพธ์ |
|---------|-----|--------|
| **สร้างรายการนับ** | `POST /api/v1/stock-counts` | SC-20260415-001, status = `open`, auto-populate lines จาก stock_levels |
| **บันทึกจำนวนจริง** | `POST /api/v1/stock-counts/:id/enter` | status = `counting`, คำนวณ variance |
| **Post ปรับสต็อก** | `POST /api/v1/stock-counts/:id/post` | status = `posted`, สร้าง stock_movements + JE |

#### ตารางผลการนับ

```
┌──────────┬──────────────┬──────────────┬──────────┐
│ Product  │ Book Qty     │ Actual Qty   │ Variance │
│          │ ยอดตามบัญชี   │ นับจริง       │ ผลต่าง   │
├──────────┼──────────────┼──────────────┼──────────┤
│ Product A│     70       │     68       │    -2    │
└──────────┴──────────────┴──────────────┴──────────┘
```

### ขั้นตอนที่ 5: GL Entry สำหรับ Variance (ผลต่างจากการนับ)

```
┌─────────────────────────────────────────────────────────────────┐
│  Journal Entry: SC-JE                                           │
│  Description: "Stock count variance: SC-20260415-001"           │
│  Status: posted                                                 │
├─────┬────────────────────────────┬──────────┬───────────────────┤
│ Line│ บัญชี (Account)            │ Dr (เดบิต)│ Cr (เครดิต)       │
├─────┼────────────────────────────┼──────────┼───────────────────┤
│  1  │ 5100 ผลต่างสินค้า          │   1,000  │                   │
│     │ (Inventory Variance Loss)  │          │ (2 ชิ้น × 500 บาท)│
│  2  │ 1300 สินค้าคงคลัง          │          │   1,000           │
│     │ (Inventory Asset)          │          │                   │
├─────┼────────────────────────────┼──────────┼───────────────────┤
│     │ รวม                        │   1,000  │   1,000           │
└─────┴────────────────────────────┴──────────┴───────────────────┘
```

**หลักการ:**
- **Variance เป็นลบ (ขาด):** Dr 5100 Variance Expense / Cr 1300 Inventory (ลดสินค้า)
- **Variance เป็นบวก (เกิน):** Dr 1300 Inventory / Cr 5100 Variance (เพิ่มสินค้า)

> **การคำนวณ:** ระบบดึง cost_price_satang จาก products table
> Variance value = |variance quantity| × unit cost = 2 × 500 = 1,000 บาท

```

│  4  │ ✅ ออก 50 ทวิ ให้พนักงานทุกคน                               │
├─────┼────────────────────────────────────────────────────────────┤
│  5  │ ✅ จัดทำงบการเงินประจำปี                                     │
│     │    • งบกำไรขาดทุน (P&L)                                     │
│     │    • งบแสดงฐานะทางการเงิน (Balance Sheet)                   │
│     │    • งบกระแสเงินสด (Cash Flow)                              │
├─────┼────────────────────────────────────────────────────────────┤
│  6  │ ✅ สร้าง Fiscal Year ใหม่                                   │
│     │    API: POST /api/v1/fiscal-years                          │
│     │    → สร้าง 12 งวดอัตโนมัติ (periods auto-generated)         │
└─────┴────────────────────────────────────────────────────────────┘
```

### ตัวอย่าง Year-End Closing JE

```
┌─────────────────────────────────────────────────────────────────┐
│  Journal Entry: JE-2026-CLOSE                                   │
│  Description: "Year-end closing entry for fiscal year 2026"     │
│  Fiscal Period: 12                                              │
├─────┬────────────────────────────┬──────────┬───────────────────┤
│ Line│ บัญชี (Account)            │ Dr (เดบิต)│ Cr (เครดิต)       │
├─────┼────────────────────────────┼──────────┼───────────────────┤
│  1  │ 4100 รายได้จากการขาย        │ 10,000   │                   │
│     │ (Close revenue)            │          │                   │
│  2  │ 5100 ต้นทุนสินค้าขาย       │          │ 25,000            │
│     │ (Close expense)            │          │                   │
│  3  │ 5200 เงินเดือนและค่าจ้าง    │          │ 30,000            │
│     │ (Close expense)            │          │                   │
│  4  │ 3200 กำไรสะสม              │ 45,000   │                   │
│     │ (Retained Earnings — loss) │          │                   │
├─────┼────────────────────────────┼──────────┼───────────────────┤
│     │ รวม                        │ 55,000   │ 55,000            │
└─────┴────────────────────────────┴──────────┴───────────────────┘
```

**หลักการ Year-End Closing:**
1. **Dr Revenue accounts** → ล้างยอด credit ของรายได้ให้เป็น 0
2. **Cr Expense accounts** → ล้างยอด debit ของค่าใช้จ่ายให้เป็น 0
3. **ผลต่าง → Retained Earnings (3200)**:
   - กำไร (Revenue > Expense): Cr 3200 (เพิ่มกำไรสะสม)
   - ขาดทุน (Expense > Revenue): Dr 3200 (ลดกำไรสะสม)

> **หมายเหตุ:** ถ้าต้องแก้ไขหลังปิดปี สามารถ reopen ได้:
> `POST /api/v1/fiscal-years/:id/reopen`
> ระบบจะสร้าง reversal JE อัตโนมัติ (swap Dr/Cr ของ closing JE)
